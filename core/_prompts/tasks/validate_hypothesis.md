Validate this implementation hypothesis against the task.

A hypothesis is approvable only if it meets these bars:

- **Root cause, not symptom.** Names what is wrong or needed.
- **Maps to concrete code.** Points at specific files, functions, or behavior.
- **Approach is viable, not speculative.** Gives an executable path.
- **Names the riskiest assumption or falsifier.** States what would disprove it.
- **Does not overclaim.** Avoids certainty beyond verified evidence.

If all bars pass, approve. Otherwise, reject and list each failing
bar with the concrete reason it fails — these become the inputs to
the next hypothesis attempt, so vague rejections waste a round.
